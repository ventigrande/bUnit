    Configure 3rd Party Libraries for Testing

    Testing a component that is built using a 3rd party component library can require additional setup steps in each test to ensure that the 3rd party components work correctly in a test context.

    For example, if a 3rd party component requires services to be registered in production code, it is likely that this is needed in a test context as well. See the Injecting Services into Components Under Test page for more info on this.

    Similarly, if a root component is needed to provide cascading values to the 3rd party components, that is likely to be needed as well. For more on this, see the Passing Parameters to Components page.
